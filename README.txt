====================================================

Сервер: 193.109.69.233 (Hostkey NL)

SNI: dzen.ru (TLS 1.3 — проверено)
Клиент: V2RayTun

====================================================
ПОРЯДОК ДЕЙСТВИЙ
====================================================

Все скрипты запускать на ЦЕЛЕВОМ сервере от root.

СПОСОБ 1 — Скопировать файлы на сервер:
  scp -r /home/claude/xray-setup/ root@193.109.69.233:/root/

СПОСОБ 2 — Копировать команды вручную из каждого скрипта.

----------------------------------------------------
ШАГ 1. Зачистка (01-cleanup.sh)
----------------------------------------------------
  bash /root/xray-setup/01-cleanup.sh

  Что делает:
  - Останавливает и удаляет все Docker-контейнеры
  - Удаляет Docker, Amnezia, Outline
  - Чистит iptables
  - Убивает всё на порту 443

  После: проверь вывод ss -tulpn
  Должны остаться только sshd (22) и systemd-resolve (53)

----------------------------------------------------
ШАГ 2. Установка Xray (02-install-xray.sh)
----------------------------------------------------
  bash /root/xray-setup/02-install-xray.sh

  Что делает:
  - Скачивает и устанавливает Xray
  - Генерирует UUID, ключи Reality, Short ID

  ВАЖНО: Запиши вывод! Тебе понадобятся:
  - UUID
  - Private Key (для сервера)
  - Public Key (для клиента)
  - Short ID

----------------------------------------------------
ШАГ 3. Конфигурация (03-configure.sh)
----------------------------------------------------
  bash /root/xray-setup/03-configure.sh <UUID> <PRIVATE_KEY> <SHORT_ID>

  Пример:
  bash /root/xray-setup/03-configure.sh \
    "a1b2c3d4-e5f6-7890-abcd-ef1234567890" \
    "WBPGjGKKez..." \
    "abcdef0123456789"

  Что делает:
  - Записывает конфиг Xray
  - Проверяет конфиг
  - Запускает Xray

----------------------------------------------------
ШАГ 4. Firewall (04-firewall.sh)
----------------------------------------------------
  bash /root/xray-setup/04-firewall.sh

  Что делает:
  - Ставит UFW
  - Открывает ТОЛЬКО 22 (SSH) и 443 (Xray)
  - Закрывает всё остальное

----------------------------------------------------
ШАГ 5. Ссылка для клиента (05-generate-link.sh)
----------------------------------------------------
  bash /root/xray-setup/05-generate-link.sh <UUID> <PUBLIC_KEY> <SHORT_ID>

  ВНИМАНИЕ: тут нужен PUBLIC Key, не Private!

  Выдаст vless:// ссылку — вставляешь её в V2RayTun.

====================================================
ПРОВЕРКА ПОСЛЕ УСТАНОВКИ
====================================================

На сервере:
  systemctl status xray          — должен быть active (running)
  ss -tulpn | grep 443           — должен слушать xray
  journalctl -u xray -n 50      — логи, если что-то не так

С телефона:
  1. Вставь ссылку в V2RayTun
  2. Подключись
  3. Открой https://whatismyipaddress.com/
  4. IP должен быть 193.109.69.233

====================================================
ЧТО ДЕЛАЕТ КОНФИГ СКРЫТНЫМ
====================================================

1. Reality — не нужен свой TLS-сертификат, используется
   настоящий сертификат dzen.ru. DPI видит валидный TLS
   к dzen.ru, а не самоподписанный мусор.

2. flow: xtls-rprx-vision — маскирует паттерны трафика,
   убирает характерные для прокси отпечатки.

3. fingerprint: chrome — TLS-отпечаток выглядит как
   обычный Chrome, а не как Go/специфичный клиент.

4. Порт 443 — стандартный HTTPS, не вызывает подозрений.

5. Никаких панелей/веб-морд — снаружи сервер выглядит
   как закрытый хост, не отвечающий ни на что кроме
   валидного Reality-хэндшейка.

6. Блокировка torrent и private IP — сервер не станет
   открытым прокси.

====================================================
ЕСЛИ НЕ РАБОТАЕТ
====================================================

1. Проверь что Xray запущен: systemctl status xray
2. Проверь логи: journalctl -u xray -n 100
3. Проверь порт: ss -tulpn | grep 443
4. Проверь firewall: ufw status
5. Проверь что ничего не конфликтует: ss -tulpn
6. С другой машины: curl -v --max-time 5 https://193.109.69.233/ -k
   (должен показать TLS handshake от dzen.ru или connection reset)
